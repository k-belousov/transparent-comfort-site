## 🏢 О проекте

"Прозрачный комфорт" — современный сайт-визитка для компании, специализирующейся на установке мягких окон для террас, кафе, загородных домов и беседок.

### Ключевые особенности:

- **9 лет опыта** на рынке Челябинска
- **3000+ реализованных объектов**
- **Гарантия до 7 лет** на все работы
- **Материалы премиум-класса** от проверенных поставщиков
- **Быстрый монтаж** за 1-2 дня
- **Полная прозрачность** ценообразования

### Целевая аудитория:

- Владельцы частных домов и коттеджей
- Владельцы кафе и ресторанов
- Представители загородных клубов
- Частные клиенты с беседками и террасами

## 🚀 Установка и запуск

### Требования:

- **Node.js** версии 18 или выше
- **npm** или **yarn** для управления зависимостями

### Локальная разработка:

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/k-belousov/transparent-comfort-landing-main.git
cd transparent-comfort-landing-main

# 2. Установите зависимости
npm install

# 3. Запустите сервер разработки
npm run dev

# 4. Откройте браузер по адресу
http://localhost:8080
```

### Сборка для продакшена:

```bash
# Создайте оптимизированную сборку
npm run build

# Проверьте сборку локально
npm run preview

# Анализ размера бандла
npm run build -- --analyze
```

### Доступные скрипты:

```json
{
  "dev": "vite",                    # Сервер разработки
  "build": "vite build",            # Продакшн сборка
  "preview": "vite preview",         # Предпросмотр сборки
  "lint": "eslint .",               # Проверка кода
  "build:dev": "vite build --mode development"  # Dev сборка
}
```

## 🌐 Развертывание

### GitHub Pages (рекомендуется):

```bash
# 1. Соберите проект
npm run build

# 2. Загрузите в GitHub
git add .
git commit -m "Deploy to GitHub Pages"
git push origin main

# 3. Активируйте Pages в настройках репозитория
# Settings → Pages → Source: Deploy from a branch → main
```

### Другие платформы:

- **Netlify**: Подключите репозиторий и настройте автоматический деплой
- **Vercel**: Импортируйте проект и разверните за 1 клик
- **Surge.sh**: `npm run build && surge dist`

Подробная инструкция в файле [`DEPLOYMENT.md`](DEPLOYMENT.md)

### Настройка Яндекс.Вебмастера (обязательно для SEO)

После развертывания сайта необходимо:

1. **Добавить сайт** в Яндекс.Вебмастер: https://webmaster.yandex.ru/
2. **Проверить права на управление** через мета-тег или файл
3. **Загрузить sitemap.xml**:
   - Яндекс.Вебмастер → Индексирование → Sitemap-файлы
   - Добавить: `/sitemap.xml`
4. **Проверить robots.txt**:
   - Яндекс.Вебмастер → Индексирование → Файл robots.txt
   - Должен отображаться без ошибок
5. **Отправить на переобход** после внесения изменений

**Важно**: Файлы `robots.txt` и `sitemap.xml` используют относительные пути, что корректно для GitHub Pages и динамических доменов.
